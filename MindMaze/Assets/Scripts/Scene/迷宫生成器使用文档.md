# MazeGenerator3D 使用文档

挂载到场景空 `GameObject` 上的 3D 迷宫生成器，基于 DFS 回溯算法。

\---

## 快速开始

1. 场景中创建空 `GameObject`，命名 `MazeGenerator`，挂载 `MazeGenerator3D.cs`
2. 准备预制体：

   * **边界围墙**：正方体/长方体，拖到 `Border Wall Prefabs`（空则 fallback 到 `Wall Prefabs`）
   * **内部墙体**：正方体/长方体，拖到 `Wall Prefabs`
   * **地板**：朝上（Y+）面片，拖到 `Floor Prefabs`
   * **天花板**：朝下（Y-）面片，拖到 `Ceiling Prefabs`（勾选 `Ceiling Faces Down`）
3. 设置参数（`Cell Size = 4`, `Wall Height = 3`, `Wall Thickness = 0.3`）
4. 运行 —— 默认 `Start()` 自动生成，也可右键组件选择 **生成迷宫** 手动触发

\---

## Inspector 参数

### 迷宫规格

|参数|类型|默认值|说明|
|-|-|-|-|
|`Width`|int|12|横向格子数|
|`Height`|int|12|纵向格子数|
|`Cell Size`|float|4|格子边长（世界单位）|
|`Wall Height`|float|3|墙体高度（Y 轴）|
|`Wall Thickness`|float|0.3|墙体厚度|
|`Prefab Unit Size`|float|1|后备尺寸：仅当预制体无 MeshFilter 时使用|
|`Floor Plane Unit Size`|float|10|地板面片原始边长。0=按 3D 模型处理。Unity Plane=10，Quad=1|
|`Ceiling Plane Unit Size`|float|10|天花板面片原始边长。0=按 3D 模型处理|
|`Ceiling Faces Down`|bool|true|天花板面片默认朝下时勾选，不额外旋转|

### 起点 / 终点

|参数|类型|默认值|说明|
|-|-|-|-|
|`Start Cell`|Vector2Int|(0,0)|起点格子坐标，自动钳制到迷宫范围内|
|`End Cell`|Vector2Int|(11,11)|终点格子坐标，自动钳制到迷宫范围内|

### 预制体数组（支持多种随机选取）

|参数|说明|
|-|-|
|`Wall Prefabs`|内部墙体预制体，每段墙随机选取一个|
|`Border Wall Prefabs`|边界围墙预制体，单独控制材质。**空则 fallback 到 Wall Prefabs**|
|`Floor Prefabs`|地板预制体，每格随机选取|
|`Ceiling Prefabs`|天花板预制体。空则不生成天花板|

### 入口 / 出口

|参数|类型|默认值|说明|
|-|-|-|-|
|`Open Entry`|bool|true|起点处是否打通外墙|
|`Open Exit`|bool|true|终点处是否打通外墙|

### 起点 / 终点房间

|参数|类型|默认值|说明|
|-|-|-|-|
|`Start Room Prefab`|GameObject|null|起点房间预制体，放置在迷宫左侧|
|`End Room Prefab`|GameObject|null|终点房间预制体，放置在迷宫右侧|
|`Room Placement Offset`|float|0|房间与迷宫边界的距离（格子数），微调位置|

### 起点 / 终点标记

|参数|类型|默认值|说明|
|-|-|-|-|
|`Start Marker Prefab`|GameObject|null|起点处的可视化标记（如旗帜）|
|`End Marker Prefab`|GameObject|null|终点处的可视化标记|
|`Marker Y Offset`|float|2|标记在 Y 轴上的偏移高度|

### 门 \& 烟雾

|参数|类型|默认值|说明|
|-|-|-|-|
|`Door Prefab`|GameObject|null|放置在通道上的门预制体|
|`Door Chance`|0\~1|0.3|每条通道生成门的概率|
|`Smoke Prefab`|GameObject|null|烟雾粒子预制体|
|`Smoke Chance`|0\~1|0.15|每条通道生成烟雾的概率|

### 装饰物 \& 灯光（可自由增删）

每个装饰条目对应一个 `DecorationEntry`：

|字段|类型|说明|
|-|-|-|
|`Prefab`|GameObject|装饰物/灯光预制体|
|`Spawn Chance`|0\~1|每个格子生成该装饰的概率|
|`Random Offset`|0\~0.5|在格子内的随机偏移幅度|
|`Uniform Scale`|float|统一缩放倍数。0=使用原始大小|
|`Random Y Rotation`|bool|是否随机 Y 轴旋转|

* `Room Decorations`：房间类装饰（如储物间），每格内随机生成
* `Object Decorations`：物体类装饰（如箱子），每格内随机生成
* `Ceiling Light Decorations`：天花板灯光，自动翻转向下
* `Wall Light Decorations`：墙壁灯光，随机选墙并自动朝内

> 装饰物/灯光自动跳过起点格和终点格。装饰物放置时自动避开墙壁区域。

### 调试寻路

|参数|类型|默认值|说明|
|-|-|-|-|
|`Enable Debug Navigation`|bool|false|启用后生成小球自动从起点走到终点|
|`Debug Ball Prefab`|GameObject|null|调试小球预制体（建议用 Sphere）|
|`Debug Ball Speed`|float|5|小球移动速度|
|`Debug Ball Y Offset`|float|1.5|小球 Y 轴偏移（离地高度）|

## 自动尺寸检测

代码在生成时读取预制体的 `MeshFilter.mesh.bounds` 获取实际网格尺寸，自动计算 `localScale`。**无需手动设 `Prefab Unit Size`**——该字段仅在没有 MeshFilter 时作为后备值。

\---

## 墙体放置规则

### 边界围墙（正方体）

四面围墙每格一段，尺寸 = `wallThickness × wallHeight × cellSize`（左右墙）或 `cellSize × wallHeight × wallThickness`（上下墙），起点/终点处自动留空：

* 左墙：X=0，沿 Z 轴排布
* 右墙：X=width × cellSize，沿 Z 轴排布
* 底墙：Z=0，沿 X 轴排布
* 顶墙：Z=height × cellSize，沿 X 轴排布

### 内部墙体（正方体）

放置在相邻格子之间，拐角处自然交叠形成实心拐角：

* 水平墙（右墙）：X=相邻边界，尺寸 = `(wallThickness, wallHeight, cellSize)`
* 垂直墙（顶墙）：Z=相邻边界，尺寸 = `(cellSize, wallHeight, wallThickness)`

### 地板 / 天花板

每格铺设，自动缩放填充。天花板放置于 `wallHeight` 高度处。

## 迷宫规模限制

迷宫生成使用 DFS 递归算法，受 C# 调用栈深度限制。建议：

|规模|状态|说明|
|-|-|-|
|≤ 50×50|安全|直接生成，无额外配置|
|50×50 \~ 150×150|可用|需确保调用栈充足（默认已够）|
|150×150 \~ 500×500|需注意|生成时间较长（数秒），递归深度触及边缘|
|> 500×500|不建议|DFS 可能栈溢出，建议改用迭代版或 BFS 生成|

> 注意：DFS 递归深度在最坏情况下（一条直线蛇形路径）等于 `width × height`，如 300×300=90000 层递归可能触发 `StackOverflowException`。实际迷宫因分支较多，通常不会达到最坏深度。

此外还需考虑性能：

* 每格生成地板、天花板、可能的内墙
* 通道数 = `width × height - 1`
* 门和烟雾按概率在每条通道上生成
* 装饰物按概率在每格内生成

以 100×100 为例（默认参数）：

* 格子数：10,000
* 通道数：9,999
* 最多生成对象：\~30,000+

## API 方法

### 生成迷宫

```csharp
// 获取组件引用
var mg = GetComponent<MazeGenerator3D>();

// 方式1：生成指定大小的迷宫，起点(0,0) 终点(w-1, h-1)
mg.GenerateMaze(8, 8);

// 方式2：生成指定起点和终点的迷宫
mg.GenerateMaze(10, 10, new Vector2Int(0, 0), new Vector2Int(9, 5));

// 方式3：使用 Inspector 中已配置的参数重新生成
mg.Generate();
```

### `CanWalk` — 格子间通行检查

```csharp
// 检查从 (0,0) 到 (0,1) 是否可以通行（无墙）
if (mg.CanWalk(0, 0, 0, 1))
    transform.position = mg.GetCellWorldPos(0, 1);
```

### `GetPassableDirections` — 获取可通行方向

```csharp
// 获取当前格子四个方向的可通行状态
// 返回 bool\[4]: { top, right, bottom, left }，true 表示可通行
bool\[] dirs = mg.GetPassableDirections(currentX, currentY);
if (dirs\[0]) Debug.Log("上方可走");      // top
//   dirs\[1] → right,  dirs\[2] → bottom,  dirs\[3] → left
```

### `FindPath` — BFS 寻路

```csharp
// 获取从起点到终点的最短路径（格子坐标列表）
var path = mg.FindPath();
if (path != null)
{
    foreach (var cell in path)
    {
        Vector3 worldPos = mg.GetCellWorldPos(cell.x, cell.y);
        Debug.Log($"途经格子: {cell} → 世界坐标: {worldPos}");
    }
}
else
{
    Debug.LogError("起点到终点不可达！");
}
```

### `GetCellWorldPos` — 格子坐标转世界坐标

```csharp
// 返回格子中心的世界坐标
Vector3 center = mg.GetCellWorldPos(3, 4);
// 格子 (3,4) 的世界坐标: X = (3+0.5)\*cellSize, Y=0, Z = (4+0.5)\*cellSize
```

### `GetCellFromWorldPos` — 世界坐标反查格子坐标

```csharp
// 将玩家当前位置反查为迷宫格子坐标
Vector2Int cell = mg.GetCellFromWorldPos(player.position);
Debug.Log($"当前在格子: {cell.x}, {cell.y}");
```

### `IsCellVisited` — 查询格子是否为走廊

```csharp
// 判断某个格子是否属于已生成的迷宫走廊
bool isPath = mg.IsCellVisited(x, y);
```

### `Width` / `Height` / `CellSize` — 只读属性

```csharp
int w = mg.Width;        // 迷宫宽度
int h = mg.Height;       // 迷宫高度
float size = mg.CellSize; // 格子大小
```

## 完整示例：玩家在迷宫中移动

```csharp
public class MazePlayer : MonoBehaviour
{
    public MazeGenerator3D maze;
    public float moveSpeed = 5f;
    private Vector2Int currentCell;

    void Start()
    {
        // 生成后获取起点位置
        maze.GenerateMaze(15, 15);
        currentCell = new Vector2Int(0, 0);
        transform.position = maze.GetCellWorldPos(currentCell.x, currentCell.y);
    }

    void Update()
    {
        Vector2Int target = currentCell;

        if (Input.GetKeyDown(KeyCode.W) || Input.GetKeyDown(KeyCode.UpArrow))
            target = new Vector2Int(currentCell.x, currentCell.y + 1);
        else if (Input.GetKeyDown(KeyCode.S) || Input.GetKeyDown(KeyCode.DownArrow))
            target = new Vector2Int(currentCell.x, currentCell.y - 1);
        else if (Input.GetKeyDown(KeyCode.D) || Input.GetKeyDown(KeyCode.RightArrow))
            target = new Vector2Int(currentCell.x + 1, currentCell.y);
        else if (Input.GetKeyDown(KeyCode.A) || Input.GetKeyDown(KeyCode.LeftArrow))
            target = new Vector2Int(currentCell.x - 1, currentCell.y);
        else
            return;

        // 检查是否可以通行
        if (maze.CanWalk(currentCell.x, currentCell.y, target.x, target.y))
        {
            currentCell = target;
            transform.position = maze.GetCellWorldPos(currentCell.x, currentCell.y);
        }
    }
}
```

## 右键菜单

挂载组件后，在 Inspector 中右键点击组件标题：

* **生成迷宫**：使用当前 Inspector 参数重新生成（覆盖原有迷宫）
* **调试寻路**：生成小球从起点走到终点，验证迷宫可达性（仅在 Play 模式下有效）

\---

## 注意事项

* 迷宫生成时先清空所有子物体再重建
* `MazeCell3D` 中 `top/right/bottom/left = true` 表示有墙
* 装饰物/烟雾/灯光自动跳过起点格和终点格
* 烟雾和门仅在通道位置（被打通的墙处）生成
* 每种数组至少放入一个预制体，否则对应结构不生成
* 边界围墙默认与内部墙体共用同一预制体（material 可通过 `Border Wall Prefabs` 单独指定）
* DFS 生成的迷宫保证从起点到终点有且仅有一条路径（完美迷宫）
* `FindPath()` 返回的是 BFS 最短路径，可用于 AI、迷你地图或提示系统
