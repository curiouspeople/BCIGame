# MazeGenerator3D 使用文档

挂载到场景空 `GameObject` 上的 3D 迷宫生成器，基于 DFS 回溯算法。

\---

## 快速开始

1. 场景中创建空 `GameObject`，命名 `MazeGenerator`，挂载 `MazeGenerator3D.cs`
2. 准备预制体：

   * **边界围墙**：一个面片（Quad），默认朝向 Z 轴正方向（前方），拖到 `Border Wall Prefabs`。代码自动旋转使其面向迷宫内部
   * **内部墙体**：由多块面片拼成的长方体（边长 4），拖到 `Wall Prefabs`
   * **地板**：朝上面片，拖到 `Floor Prefabs`
   * **天花板**：朝下面片，拖到 `Ceiling Prefabs`（勾选 `Ceiling Faces Down`）
3. 设置参数（`Cell Size = 4`, `Wall Height = 3`, `Wall Thickness = 0.3`）
4. 运行

\---

## Inspector 参数

### 迷宫规格

|参数|类型|默认|说明|
|-|-|-|-|
|`Width`|int|12|横向格子数|
|`Height`|int|12|纵向格子数|
|`Cell Size`|float|4|格子边长（世界单位）|
|`Wall Height`|float|3|墙体高度|
|`Wall Thickness`|float|0.3|内部墙体厚度|
|`Prefab Unit Size`|float|1|后备尺寸：仅当预制体无 MeshFilter 时使用|
|`Floor Plane Unit Size`|float|10|地板面片边长。0=按3D处理。Plane=10，Quad=1|
|`Ceiling Plane Unit Size`|float|10|天花板面片边长。0=按3D处理|
|`Ceiling Faces Down`|bool|true|天花板面片默认朝下时勾选|

### 预制体数组

|参数|说明|
|-|-|
|`Wall Prefabs`|内部墙体（长方体预制体），随机选取|
|`Border Wall Prefabs`|边界围墙（面片预制体），空则 fallback 到 Wall Prefabs|
|`Floor Prefabs`|地板（朝上面片），随机选取|
|`Ceiling Prefabs`|天花板（朝下面片），空则不生成|

### 其他参数

`Open Entry` / `Open Exit`、`Start/End Cell`、`Door Prefab`、`Smoke Prefab`、装饰物、灯光、调试寻路等均与之前相同。

\---

## 自动尺寸检测

代码在生成时读取预制体的 `MeshFilter.mesh.bounds` 获取实际网格尺寸，自动计算 `localScale`。**无需手动设 `Prefab Unit Size`**——该字段仅在没有 MeshFilter 时作为后备值。

\---

## 墙体放置规则

### 边界围墙（面片）

四面围墙各用一个面片，法线朝向迷宫内部。每格逐段生成，起点/终点房间处自动留空。面片无厚度，紧贴边缘：

* 左墙 X=0，朝 +X
* 右墙 X=width\*cellSize，朝 -X
* 底墙 Z=0，朝 +Z
* 顶墙 Z=height\*cellSize，朝 -Z

### 内部墙体（长方体）

放置在相邻格子之间，不裁剪拐角。两面垂直墙体在拐角处自然交叠，形成实心拐角。

\---

## 装饰物避墙

装饰物放置时自动避开墙壁区域——根据格子四面墙的存在情况，限制偏移范围使装饰物不进入墙体内部。

\---

## API 方法

与之前相同：`GenerateMaze(w,h)`、`CanWalk`、`GetPassableDirections`、`GetCellWorldPos`、`GetCellFromWorldPos`、`FindPath` 等。

\---

## 注意事项

* 迷宫生成时先清空子物体再重建
* `MazeCell3D` 中 `top/right/bottom/left = true` 表示有墙
* 装饰物/烟雾自动跳过起点和终点格子
* 烟雾仅在通道位置生成
* 每种数组至少放入一个预制体
* 边界围墙面片默认应朝前（Z+），代码自动旋转到正确朝向
