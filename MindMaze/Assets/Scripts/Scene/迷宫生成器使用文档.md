# MazeGenerator3D 使用文档

挂载到场景空 `GameObject` 上的 3D 迷宫生成器，基于 DFS 回溯算法。

---

## 快速开始

1. 在场景中创建空 `GameObject`，命名为 `MazeGenerator`
2. 挂载 `Scripts/Scene/MazeGenerator3D.cs`
3. 准备墙体、地板、天花板的预制体（每种可准备多个变体）
4. 拖入 Inspector 对应数组槽位，运行即可

---

## Inspector 参数

### 迷宫规格
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `Width` | int | 12 | 迷宫横向格子数 |
| `Height` | int | 12 | 迷宫纵向格子数 |
| `Cell Size` | float | 4 | 每个格子的边长（世界单位） |
| `Wall Height` | float | 3 | 墙体高度 |
| `Wall Thickness` | float | 0.3 | 墙体厚度 |
| `Prefab Unit Size` | float | 1 | **预制体基础边长**。代码以 1×1×1 为标准 Cube 计算所有生成物体的缩放。如果你的预制体默认是 2×2×2，设为 2 即可自动等比缩放 |

### 起点 / 终点
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `Start Cell` | Vector2Int | (0, 0) | 迷宫起点格子坐标 |
| `End Cell` | Vector2Int | (11, 11) | 迷宫终点格子坐标 |

### 墙体 / 地板 / 天花板预制体（数组，随机选取）
| 参数 | 类型 | 说明 |
|------|------|------|
| `Wall Prefabs` | GameObject[] | 墙体预制体数组，每面墙随机选取一种。拖动 Size 后填入多个变体 |
| `Floor Prefabs` | GameObject[] | 地板预制体数组，每个格子随机选取一种 |
| `Ceiling Prefabs` | GameObject[] | 天花板预制体数组，每个格子随机选取一种（空则不生成天花板） |

### 入口 / 出口
| 参数 | 说明 |
|------|------|
| `Open Entry` | 是否打通起点处的外墙 |
| `Open Exit` | 是否打通终点处的外墙 |

### 起点 / 终点房间
| 参数 | 说明 |
|------|------|
| `Start Room Prefab` | 起点房间预制体，放置在迷宫起点左侧。房间右墙应留门洞与迷宫入口对齐 |
| `End Room Prefab` | 终点房间预制体，放置在迷宫终点右侧。房间左墙应留门洞与迷宫出口对齐 |
| `Room Placement Offset` | 房间与迷宫边界的微调距离（单位：格子数，默认 0 即紧贴） |

### 起点 / 终点可视化标记
| 参数 | 说明 |
|------|------|
| `Start Marker Prefab` | 起点上方悬浮的标记预制体（如旗帜、光柱） |
| `End Marker Prefab` | 终点上方悬浮的标记预制体 |
| `Marker Y Offset` | 标记离地高度（默认 2） |

### 安全锁控门
| 参数 | 说明 |
|------|------|
| `Door Prefab` | 门预制体（可为 null） |
| `Door Chance` | 每个通道口生成门的概率（0~1） |

### 烟雾隐蔽通道
| 参数 | 说明 |
|------|------|
| `Smoke Prefab` | 烟雾粒子预制体（可为 null） |
| `Smoke Chance` | 每个通道口生成烟雾的概率（0~1） |

### 房间类装饰（可自由增删）
| 参数 | 类型 | 说明 |
|------|------|------|
| `Room Decorations` | List\<DecorationEntry\> | Inspector 中点击 `+` 添加条目。适用于无意义房间、储物间等。默认放置在地板 |

### 物体类装饰（可自由增删）
| 参数 | 类型 | 说明 |
|------|------|------|
| `Object Decorations` | List\<DecorationEntry\> | Inspector 中点击 `+` 添加条目。适用于储水罐、箱子等。默认放置在地板 |

### 灯光（可自由增删）
| 参数 | 类型 | 说明 |
|------|------|------|
| `Ceiling Light Decorations` | List\<DecorationEntry\> | 天花板灯光，自动翻转 180° 朝下，Inspector 中可增减条目 |
| `Wall Light Decorations` | List\<DecorationEntry\> | 墙壁灯光，随机选一面墙自动朝向格内，Inspector 中可增减条目 |

每条 `DecorationEntry` 包含:

| 子参数 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `Prefab` | GameObject | null | 装饰物/灯光预制体 |
| `Spawn Chance` | float (0~1) | 0.1 | 每个格子生成概率 |
| `Random Offset` | float (0~0.5) | 0.3 | 格子内随机位置偏移（墙壁灯光为沿墙面偏移） |
| `Uniform Scale` | float | 1 | 统一缩放倍数（相对于 Prefab Unit Size）。0 则保留预制体原始大小 |
| `Random Y Rotation` | bool | true | 是否随机 Y 轴旋转（灯光类会自动修正朝向后再随机） |

### 调试寻路
| 参数 | 说明 |
|------|------|
| `Enable Debug Navigation` | 勾选后生成迷宫时自动运行调试小球 |
| `Debug Ball Prefab` | 小球预制体（推荐 Unity 默认 Sphere） |
| `Debug Ball Speed` | 小球移动速度（默认 5） |
| `Debug Ball Y Offset` | 小球离地高度（默认 1.5） |

## API 方法

### 生成迷宫

```csharp
// 生成指定尺寸迷宫（起点固定 (0,0)，终点 (w-1, h-1)）
mazeGenerator.GenerateMaze(5, 5);

// 生成指定尺寸和起点终点的迷宫
mazeGenerator.GenerateMaze(5, 5, new Vector2Int(0, 0), new Vector2Int(4, 2));

// 使用 Inspector 当前参数重新生成
mazeGenerator.Generate();
```

### 移动接口（供玩家移动脚本调用）

迷宫的坐标系：`x` 从左到右，`y` 从下到上。方向定义如下：

```
          y+ (上)
           |
           |
   x- -----+----- x+
  (左)     |     (右)
           |
           |
          y- (下)
```

---

#### `CanWalk(int fromX, int fromY, int toX, int toY)`

判断**相邻两个格子**之间是否有墙阻挡。两个格子必须紧邻（上下左右），不能斜走。

```csharp
MazeGenerator3D maze = GetComponent<MazeGenerator3D>();

// 从 (0,0) 往右走一步到 (1,0) —— 两格紧邻，x 相差 1，y 相同
if (maze.CanWalk(0, 0, 1, 0))
    Debug.Log("右方无墙，可通行");

// 从 (3,2) 往上走到 (3,3)
if (maze.CanWalk(3, 2, 3, 3))
    Debug.Log("上方无墙，可通行");

// 错误示例：斜走 (0,0) → (1,1) —— 会返回 false
maze.CanWalk(0, 0, 1, 1);  // false，因为 x 和 y 同时变了

// 错误示例：越界 —— 会返回 false
maze.CanWalk(-1, 0, 0, 0); // false，坐标超出迷宫范围
```

返回值 `true` 的条件（全部满足才成立）：
1. `from` 和 `to` 的曼哈顿距离为 1（即只有一个轴相差 1）
2. 两个格子坐标都在迷宫范围内
3. 两格之间的**两个方向**均无墙（`from` 的出口方向 + `to` 的入口方向）

---

#### `GetPassableDirections(int x, int y)`

查询某个格子四个方向哪些能走。返回 `bool[4]`：

| 索引 | 方向 | 含义 |
|------|------|------|
| `[0]` | 上 (top) | 上方格子存在且无墙 → `true` |
| `[1]` | 右 (right) | 右方格子存在且无墙 → `true` |
| `[2]` | 下 (bottom) | 下方格子存在且无墙 → `true` |
| `[3]` | 左 (left) | 左方格子存在且无墙 → `true` |

```csharp
bool[] dirs = maze.GetPassableDirections(2, 3);

// 如果 dirs[0] 为 true，说明可以从 (2,3) 走到 (2,4)
// 如果 dirs[2] 为 false，说明 (2,3) 下方是墙或已到边界

if (dirs[0]) Debug.Log("上方可走");
if (dirs[1]) Debug.Log("右方可走");
if (dirs[2]) Debug.Log("下方可走");
if (dirs[3]) Debug.Log("左方可走");
```

### 坐标工具

```csharp
// 格子 (x, y) 的世界坐标中心
Vector3 pos = maze.GetCellWorldPos(2, 3);

// 世界坐标反查格子坐标 —— 玩家知道自己脚下是哪个格子
Vector2Int cell = maze.GetCellFromWorldPos(transform.position);

// 格子是否已访问（属于迷宫走廊）
bool visited = maze.IsCellVisited(2, 3);
```

### 寻路

```csharp
// 获取起点到终点的 BFS 最短路径，返回格子坐标列表
// 不可达时返回 null
List<Vector2Int> path = mazeGenerator.FindPath();
```

### 只读属性

```csharp
int w = mazeGenerator.Width;         // 迷宫宽度
int h = mazeGenerator.Height;        // 迷宫高度
float size = mazeGenerator.CellSize; // 格子大小
```

## 调试寻路

两种方式触发：

1. **自动触发**：Inspector 勾选 `Enable Debug Navigation` 并拖入球体预制体，运行时自动执行
2. **手动触发**：运行时右键组件 → `调试寻路`，或在代码中调用：
   ```csharp
   StartCoroutine(mazeGenerator.DebugNavigate());
   ```
   小球会沿 BFS 最短路径从起点移动到终点，Console 输出步数和结果。

## 装饰物配置示例

在 Inspector 中配置四类装饰物：

```
房间类装饰 (Room Decorations):
  [+ 添加条目] → Prefab: 储物间预制体, Spawn Chance: 0.05
  [+ 添加条目] → Prefab: 废弃房间预制体, Spawn Chance: 0.03

物体类装饰 (Object Decorations):
  [+ 添加条目] → Prefab: 储水罐预制体, Spawn Chance: 0.1
  [+ 添加条目] → Prefab: 箱子预制体,   Spawn Chance: 0.08

天花板灯光 (Ceiling Light Decorations):
  [+ 添加条目] → Prefab: 顶灯预制体, Spawn Chance: 0.3
  [+ 添加条目] → Prefab: 吊灯预制体, Spawn Chance: 0.1

墙壁灯光 (Wall Light Decorations):
  [+ 添加条目] → Prefab: 壁灯预制体, Spawn Chance: 0.15
```

### 各类别行为

| 列表 | 放置位置 | 朝向 | 偏移 |
|------|----------|------|------|
| 房间/物体装饰 | y=0（地面） | 可选随机 Y 旋转 | 格子内 XZ 随机偏移 |
| 天花板灯光 | y≈wallHeight（天花板） | 自动翻转 180° 朝下，可选叠加随机 Y 旋转 | 格子内 XZ 随机偏移 |
| 墙壁灯光 | y=wallHeight×0.5（半墙高） | 自动朝向格子内侧 | 沿墙面随机滑动偏移 |

每个条目独立计算概率，一个格子可能同时生成多个装饰物/灯光。
所有装饰物自动跳过起点/终点格子。

## 起点/终点房间 & 标记

### 物体尺寸基准

代码中所有生成物体的 `localScale` 都以 **1×1×1 的标准 Unity Cube** 为基准计算：

```
实际缩放 = 目标尺寸 ÷ Prefab Unit Size
```

- 墙体厚度 `wallThickness=0.3` → 墙体 X 轴缩放为 `0.3 / prefabUnitSize`
- 地板边长 `cellSize=4` → 地板 XZ 缩放为 `4 / prefabUnitSize`
- 装饰物 `uniformScale=1` → 整体缩放为 `1 / prefabUnitSize`

因此如果你的预制体是 2×2×2 的模型，把 `Prefab Unit Size` 设为 `2`，所有物体就会自动等比缩放成正确的世界尺寸。

### 房间与迷宫的连通

起点房间放置在迷宫 **左侧**，终点房间放置在 **右侧**：

```
[起点房间]  ||门||  [迷宫 5×5]  ||门||  [终点房间]
    ← 左侧     入口(0,0)         出口(4,4)     右侧 →
```

- **起点房间**：制作预制体时，房间的**右墙**留出门洞。代码将其放在起点格左侧 `cellSize + roomPlacementOffset` 处，房间门洞自然对齐迷宫的入口开口
- **终点房间**：制作预制体时，房间的**左墙**留出门洞。代码将其放在终点格右侧，出口开口对齐房间门洞
- 通过 `Room Placement Offset` 可微调房间与迷宫边界的间距

这样就实现了：**起点房间 → 穿过一扇门 → 迷宫全图 → 穿过一扇门 → 终点房间**。

### 可视化标记

拖入任意预制体到 `Start Marker Prefab` / `End Marker Prefab`（如旗帜、光柱、箭头等），运行时会在起点/终点格子上方 `markerYOffset` 高度处生成，方便观察和调试。

## 玩家逐格移动示例

```csharp
public class PlayerGridMovement : MonoBehaviour
{
    public MazeGenerator3D maze;
    private Vector2Int currentCell;

    private void Start()
    {
        // 根据出生位置反查所在格子
        currentCell = maze.GetCellFromWorldPos(transform.position);
        transform.position = maze.GetCellWorldPos(currentCell.x, currentCell.y);
    }

    private void Update()
    {
        Vector2Int target = currentCell;

        if (Input.GetKeyDown(KeyCode.W)) target.y += 1;
        if (Input.GetKeyDown(KeyCode.S)) target.y -= 1;
        if (Input.GetKeyDown(KeyCode.D)) target.x += 1;
        if (Input.GetKeyDown(KeyCode.A)) target.x -= 1;

        if (target != currentCell && maze.CanWalk(currentCell.x, currentCell.y, target.x, target.y))
        {
            currentCell = target;
            Vector3 pos = maze.GetCellWorldPos(currentCell.x, currentCell.y);
            pos.y = transform.position.y;
            transform.position = pos;
        }
    }
}
```

也可以配合 `GetPassableDirections` 做 UI 提示：

```csharp
// 随时获取玩家当前所在的格子
Vector2Int cell = maze.GetCellFromWorldPos(transform.position);

// 查询当前格子四个方向哪些能走（用于 UI 箭头提示）
bool[] dirs = maze.GetPassableDirections(cell.x, cell.y);
upArrow.SetActive(dirs[0]);    // 上方可走
rightArrow.SetActive(dirs[1]); // 右方可走
downArrow.SetActive(dirs[2]);  // 下方可走
leftArrow.SetActive(dirs[3]);  // 左方可走
```

## 注意事项

- 迷宫生成时会先清空子物体再重建，不要将其他逻辑物体挂在生成器下
- `MazeCell3D` 中 `top/right/bottom/left = true` 表示**有墙**，`false` 表示**通道**
- 装饰物生成会自动跳过起点格和终点格
- 每种墙体/地板/天花板数组至少放入一个预制体，否则对应部分不会生成
- 调试小球在到达终点后 0.5 秒自动销毁
- 编辑器模式下右键组件菜单可直接触发 `生成迷宫` 和 `调试寻路`
